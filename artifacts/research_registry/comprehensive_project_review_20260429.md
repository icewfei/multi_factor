## 4. 关键偏离项评估

### 4.1 α alpha158 大因子池整包进入 🟡 中等风险

**框架原文**: "Alpha158等大因子池不得整包进入主线候选池"

**实际**: Round 11b 明确以 qlib Alpha158 全 158 信号为目标执行批量扫描。36 个 alpha158_full 信号已完成诊断，另有 29 个 alpha158 命名信号。

**风险**: 
- 全因子导入大幅增加搜索空间 → 伪发现概率上升
- alpha158 的 qlib 定义与项目 native 实现可能存在口径差异
- 全量扫矿行为违背"先理解再优化"的框架精神

**缓解**: 
- 5 个 alpha158 信号入围正向，说明该池有一定含金量
- 但如果用于确认性研究，必须严格遵守单维改动和预注册

### 4.2 测试集可见性 🔴 高风险

**当前 snapshot `warehouse_20260418_181408` 包含截至 2026-04-10 的全部数据。**

所有的 67 轮研究、323 个候选、286 个信号诊断，都在同一个 snapshot 上运行，该 snapshot 包含测试集区间（2022-2025）。

**后果**: 
- 所有"探索性诊断"实际上已经看到了测试集的数据
- 如果不处理这个事实，后续任何"确认性研究"的测试集评估都无法声称独立
- **要么承认当前所有结果已消耗了样本外资格，要么回溯划分一个不含测试集的 snapshot**

**这是项目当前最严重的治理问题。**

### 4.3 无确认性研究 🟡 中等风险

67 轮研究全部是 exploratory。没有一轮满足"预注册 + 冻结候选列表 + 冻结通过标准 + 单维改动"的确认性研究要求。

框架要求主线晋升前必须走完确认性研究审核，这个要求尚未启动。

### 4.4 系统自检跳过 🟡 中等风险

框架 14.3 要求的系统自检（安慰剂/反证测试）已完成脚本但未正式执行。

虽然有 25 个正向信号，但在没有对标 shuffle baseline 的情况下，"正向"的含义不够严格。

### 4.5 tradability 独立审计未完成 🟢 低风险

tradability、标签、样本资格矩阵以 shared contract 形式存在于共享数据源中。虽然未作为独立模块“正式验收”，但已在 serving 视图中可用，且 27 次 fullchain 运行实际上消费了这些表。

## 8. 长期路径校准建议

当前项目在研究方式上存在一个系统性问题——**研究超前于制度**。框架的设计是先建立可信基础设施 → 再做有限探索 → 再确认 → 再执行。但实际推进中，基础设施建了一半就进入了大规模探索。

建议后续遵守以下纪律：

1. **每个研究周期只回答一个清晰问题**（框架 14.7），而不是同一时期并行数十个信号诊断
2. **确认性研究开始后，不再潜入新的候选因子**（框架 14.7 正式纪律）
3. **每周形成一个正式结论**: KEEP / INCONCLUSIVE / REJECT（框架 14.8）
4. **正式结果查看顺序**: 先看数据质量 → 覆盖率 → 单调性 → 一致性 → 流动暴露 → 最后看收益指标（框架 14.9）
