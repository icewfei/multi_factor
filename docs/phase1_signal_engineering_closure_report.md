# 第一阶段信号工程收口报告

## 收口结论

第一阶段的核心结论是：

- 当前日频量价信号工程阶段已经收口
- 最佳探索性候选为 `multi_equal_weight_v1`
- 严格确认性赢家为 `none`
- 下一阶段若要继续推进，需要新的数据模态或项目范围变化

这一结论直接来自 [项目状态总结 2026-05-07](/Users/wy/MiscProject/multi_factor/项目总纲及计划/project_status_20260507.md) 与现有总纲类文档，而不是本轮新增分析。

## 已探索方向

根据项目状态文档，2026-05-06 至 2026-05-07 已完成最后一轮系统性的信号工程探索，收口方向包括：

- 隔夜/日内收益拆解与反转交互
- decile 非线性映射
- power transform 系列
- 行业/市值中性化
- 中性化与尾部切除组合
- 多信号 IC 加权或等权复合

这些方向是在相同 trainval-only 研究快照、相同执行语义和相同核心标签目标附近完成的局部深挖。相关背景也见 [note_local_feature_space_exhaustion_20260506.md](/Users/wy/MiscProject/multi_factor/项目总纲及计划/note_local_feature_space_exhaustion_20260506.md)。

## 为什么没有严格确认性赢家

当前没有严格确认性赢家，不是因为完全没有任何改善，而是因为改善没有转化为满足门槛的严格确认性结果。

现有文档给出的关键信息是：

- `multi_equal_weight_v1` 是探索性赢家
- 其若干统计指标优于旧基线
- 但年化相对收益仍为负
- 因此 `strict_confirmatory_winner = none`

这说明第一阶段的真实情况是：

- 项目并没有陷入“所有尝试都完全无改善”的状态
- 但这些改善没有跨过严格确认性晋升所需的收益边界
- 因此不能把探索性优胜者包装成正式赢家

## Oracle 结果说明了什么

根据 [project_status_20260507.md](/Users/wy/MiscProject/multi_factor/项目总纲及计划/project_status_20260507.md)、[framework_method_note.md](/Users/wy/MiscProject/multi_factor/项目总纲及计划/framework_method_note.md) 与 [self_audit_20260501.md](/Users/wy/MiscProject/multi_factor/项目总纲及计划/self_audit_20260501.md)：

- 修正后的 Oracle 在同一执行语义和成本模型下可以产生极大的正收益
- 因此“框架被执行层完全封死”这一旧叙事不能继续成立
- 这并不意味着真实信号已经足够强
- 它只说明当前框架在该 Oracle diagnostic 下不是 execution-blocked

在审计上，应把这层含义写清楚：

- Oracle 证明框架不是被执行语义机械封死
- Oracle 不等于真实可学习信号已经足够
- Oracle 也不自动授予任何现实候选以赢家资格

## 当前瓶颈位于信号层

项目状态文档已明确指出：

- 所有真实信号的年化相对收益均为负
- Oracle 在同一框架下显著为正
- 差距主要来自信号层

因此，本阶段的主导性解释已经从旧的 execution-blocked 叙事，转向：

- 当前日频量价信息源下，真实可提取预测信息不足
- 经过现有组合与局部工程后，仍不足以覆盖交易成本

这也是为何当前结论更接近“信息源瓶颈”而不是“执行语义瓶颈”。

## 为什么暂停继续日频量价信号探索

暂停的原因不是研究流程中断，而是因为阶段性收口条件已经具备：

- 同一数据模态下已经完成多轮系统性搜索
- 多类局部变换与组合已被尝试
- 探索性赢家已出现，但严格确认性赢家仍为空
- Oracle 进一步把领先问题抬升到 signal/learnability 层
- 继续在同一局部特征空间内做相似变体，预期收益递减

因此，当前更合理的动作是：

- 冻结继续自由探索日频量价信号
- 固化现有证据与边界
- 为新的数据模态或新的项目范围重启做准备

## 本阶段收口后保留的有效资产

第一阶段并非“失败后清空”，而是留下了可复用资产：

- 研究脚本集合
- 数据契约与结果 Schema
- 已存在的运行态与审计产物
- 探索性赢家与失败证据
- 对执行语义、Oracle 解释和证据口径的更清晰边界

这些资产构成下一阶段重启时的工程与治理基础。
