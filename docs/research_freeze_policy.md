# 研究冻结政策

## 目的

本文档记录第一阶段信号工程收口后的冻结边界，防止在相同数据模态、相同 trainval-only 结果上继续发生无边界的自由探索或叙事漂移。

## 冻结事项

### 1. 冻结继续自由探索日频量价信号

在当前阶段，冻结继续基于同一批日频量价特征空间做新的自由探索，包括但不限于：

- 新增局部变体
- 新增同类窗口组合
- 新增仅在既有价量字段上做轻微变形的候选

冻结理由：

- 第一阶段已经完成系统性搜索
- 探索性赢家已出现
- 严格确认性赢家仍为空
- 继续局部搜索的边际信息价值已经显著下降

### 2. 冻结基于 trainval readout 调参

冻结依据 trainval-only readout 继续进行：

- 模型参数微调
- 阈值回抠
- 排名方向翻修后重新包装
- 组合权重为追逐 trainval 指标而做的后验修饰

理由是当前 trainval 结果已经承担了研究决策作用，再沿此继续调参会进一步稀释证据等级。

### 3. 冻结把负相对收益候选包装为赢家

当前允许保留“探索性赢家”这一术语，但冻结以下叙事：

- 把年化相对收益为负的候选包装成严格确认性赢家
- 用局部统计指标改善替代正式收益结论
- 用“比旧基线好一些”来替代“达到正式晋升标准”

现阶段唯一允许的精确表述应是：

- `exploratory_winner = multi_equal_weight_v1`
- `strict_confirmatory_winner = none`

### 4. 冻结用 `fixed_test` 命名 trainval 结果作为对外胜利证据

历史目录名中的 `fixed_test` 不在本轮改名，但从当前开始冻结以下做法：

- 用 `fixed_test` 命名自动暗示结果是独立 OOS 证据
- 在未说明 trainval-only 绑定关系的情况下引用这些结果
- 用历史命名掩盖证据层级问题

从现在起，凡是引用此类结果，都应同时说明：

- 它是否绑定 `warehouse_20260429_trainval_20211231`
- 它是否属于 trainval-only 研究期证据
- 它是否已被阅读并参与后续研究决策

## 解冻条件

以下情况可触发新阶段立项，而不是解除旧阶段冻结：

- 引入新的数据模态
- 改变项目范围或研究问题
- 建立新的严格隔离评估层
- 对现有工程做模块化、测试化、审计化整理

## 本政策不冻结的事项

本政策不冻结以下工作：

- 补齐 `README/docs/src/tests/configs` 审计骨架
- 梳理已有研究结论
- 固化契约、Schema 与阶段文档
- 为下一阶段数据模态扩展做准备

## 总结

当前冻结政策的核心不是“停止一切工作”，而是：

- 停止继续消耗同一批 trainval 证据做自由探索
- 停止把探索性改善误写成严格确认性成功
- 把项目从研究扩张阶段切换到收口、固化和边界治理阶段
